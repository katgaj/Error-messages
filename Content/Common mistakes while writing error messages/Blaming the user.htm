Blaming the user

Never blame the user and do not use condescending language. Users are already frustrated when they get an error message.This means you should avoid using phrases like “you did” or “you didn’t” when explaining what went wrong.You create the product, so you are responsible for any mistakes that the user will encounter.

The user is already frustrated, do not tell them they made a mistake. The better option is to acknowledge that the problem evoked on your side, which is, in fact, true. It is your job to work on the product to eliminate as many error-causing scenarios as possible.
