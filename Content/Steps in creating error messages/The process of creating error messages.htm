The process of creating error messages

From the technical point of view, follow the steps to make your work on error messages efficient:

  1. Create a spreadsheet, for example, using Google Spreadsheets so you can share it online.

  2. Add columns for the error message and the root cause.

  3. Give each error a number so you can track them.

  4. Establish an error message quality-control team.

  5. Find best practices. Share with the team.

  6. Create simple to follow writing guidelines for the error messages. Share examples.

  7. Include error and information messages in the design phase.

  8. Place the messages under source control.

  9. Review messages during the development lifecycle.

  10. Attempt to eliminate the need for messages.

  11. Carry out acceptance tests.

  12. Review and revise messages periodically.
